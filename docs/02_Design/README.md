# Design

## Purpose
This directory contains system architectures, cognitive module specifications, design patterns, and technical blueprints for the CSAL project.

## Contents
- System architecture documents
- Cognitive module specifications
- Design patterns and libraries
- UML diagrams and flowcharts
- Data flow diagrams
- Integration patterns
- Architecture decision records (ADRs)

## AI Tool Usage
**When to use:** For understanding system structure, planning implementations, and making architectural decisions.

**Embedding priority:** CRITICAL - Primary reference for all implementation work.

**Chunking strategy:** By module or subsystem, with separate chunks for interfaces, data flows, and patterns.

**Metadata tags to include:**
- `document_type: design`
- `module: [specific cognitive module]`
- `cognitive_function: [function implemented]`
- `ai_pattern: [architectural pattern used]`
- `dependencies: [list of dependent modules]`
- `brain_analogue: [corresponding brain region/network]`

## Document Standards
All design documents should:
1. Include clear diagrams (Mermaid, PlantUML, or images)
2. Specify interfaces and APIs
3. Document data structures and schemas
4. Explain design rationale
5. Note brain-inspired design principles
6. Include example use cases
