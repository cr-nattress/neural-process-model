# Program Management

## Purpose
This directory contains project roadmaps, milestones, governance documents, team roles, and organizational materials.

## Contents
- Project roadmap
- Milestone definitions and tracking
- Governance model
- Team roles and responsibilities
- Meeting notes and decisions
- Resource allocation
- Risk management
- Collaboration guidelines

## AI Tool Usage
**When to use:** For understanding project priorities, timelines, and organizational structure.

**Embedding priority:** LOW - Primarily human-oriented organizational info.

**Chunking strategy:** By milestone or governance topic.

**Metadata tags to include:**
- `document_type: program`
- `category: [roadmap|governance|roles|resources]`
- `stage: [0|1|2|3|4|5]` (referring to 5-stage evolution plan)
- `priority: [critical|high|medium|low]`

## Document Standards
All program documents should:
1. Be clear and actionable
2. Include timelines and deadlines
3. Assign clear responsibilities
4. Track progress and status
5. Document decision-making processes
6. Maintain version history
